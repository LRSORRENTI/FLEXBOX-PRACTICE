So what is flexbox?

The CSS flexible box module aka FLEXBOX is a one-dimensional layout model that makes it easy to:

  • Design flexible and efficient layouts
  • Distribute space among items
  • Control their alignment in a given container

Before flexbox there were four layout modes:

  • BLOCK - For sections on a webpage
  • Inline - For text
  • Table - For two-dimensional table data
  • Positioned - For explicit position of an element

But these layouts did not provide enough flexibility, we used to have to use all sorts of 'hacks' to get the layout the way we wanted, working with FLOATS, trying to VERTICALLY ALIGN the content, creating EQUAL HEIGHT COLUMNS, kind've a nightmare

BOOTSTRAP is build on top of the flex layout, and FLEXBOX itself is crucial to know for front-end developers in order to style pages responsively

Why flexbox?

  • Lot's of flexibility
  • Arrange items more efficiently
  • Provide superior spacing
  • Help maintain proper alignment and order of items

TERMINOLOGY:

When we talk about flexbox, we mainly mean two entities:

  • FLEX-CONTAINER: Our main flex space
  • FLEX-ITEMS: The immediate children elements of the flex container

For ex: if we have 'div class='container' and inside we have < div > item 1 < / div > the 'item 1' is the flex-item, and the div class is our flex container

Now we also deal with FLEXBOX AXES

  • MAIN AXIS, or X AXIS WHICH RUNS LEFT TO RIGHT ON OUR SCREEN
  • CROSS AXIS, or Y AXIS WHICH RUNS TOP TO BOTTOM ON OUR SCREEN

The starting point of the main axis is titled: MAIN START

The end point of the main axis is titled: MAIN END

And the length from the main start to the main end is called: MAIN SIZE

So in general we say that flex items FLOW from main start to main end and take up the main size as the length

Similarly with the cross axis we have the top which is called: CROSS START

The bottom of the axis called: CROSS END

And the distance between them is known as the: CROSS SIZE

Understanding flexbox is getting familiar with the properties associated with the flex container and the flex items, it's like learning anything, learn what properties exist, and how they work / fit together

So let's now go over all the properties that can be applied to a flex container:

  • DISPLAY: The display property defines the flex container, and is mandatory to work with flexbox
  • FLEX-DIRECTION: This defines the direction in which the flex-items are placed within the container
  • FLEX-WRAP: This is used to control the wrapping of items in a container
  • FLEX-FLOW: This is used as a shortcut for combining FLEX-WRAP & FLEX-DIRECTION
  • JUSTIFY CONTENT: Defines the alignment of the items along the main (x) axis
  • ALIGN-ITEMS: Defines how items are laid out along the cross (y) axis
  • ALIGN-CONTENT: This is similar to justify-content, the difference being this will align along the cross axis, instead of the main axis, BUT ALIGN-CONTENT ONLY WORKS WHEN THERE ARE MULTIPLE ROWS OF FLEX-ITEMS IN THE CONTAINER

The next property we'll practice is flex direction, the flex direction establishes the main axis, which in turn establishes how the flex items are placed inside the flex container

FLEX-WRAP

Another very useful property on our container is: FLEX-WRAP, by default, all items inside of the flex container will try to fit in a single line, if there's not enough space the items will overflow

So with flex-wrap, we have 3 possible values: nowrap, wrap, and wrap reverse.

With nowrap, nothing will change, as the browswer window shrinks the container items will simply stay where they are, and the shrunken window will obfuscate them, but if we add 'wrap' the items will auto-sort inside our container, and wrap-reverse does the same, except in the opposite direction

This also works with column, but it will shift our items into a side-by-side column structure

FLEX-FLOW

Flex-flow is shorthand for using both flex-direction, and flex-wrap at once, so flex-flow combines the two into one: flex-flow, and by default flex-flow is set to 'row' and 'nowrap'

So if we set flex-flow: row-reverse wrap ; it will combine the two, it's just a way of combining the two, if we did flex-direction: row, and flex-wrap: wrap, the exact same result would be achieved

FLEX-ITEM PROPERTY: ORDER

Flex-items appear in the order they are written in the source code, this can be changed with the order property, we can use order on a specified item in our css, if we have item5 in a list of 9 items, and we give it order: 1, it gets pushed to the end, this is because all of the other 8 items have a default order set to 0, and 0 is less than 1, so item 5 now appears after item9, and item 4 goes directly to item 6 because of the change

The order property takes an integer value, and changes the order of the items in the flex container

FLEX-ITEM PROPERTY: FLEX GROW

The flex property specifies what amount of space within the flex container the item should take up if necessary, and the flex grow factor is always relative to the size of the other items in the flex container

This is often useful when you have extra space inside of a given container, and you want the items to GROW to take up the remaining space

By default all flex-items have a flex-grow value set to 0, this is why they won't take up extra space if the container has any

Also if we use flex-grow on all of our items, the extra space will be evenly distributed among the flex items

We can also set flex-grow to different sizes on each item if we'd like them to take up different amounts of space within our container

FLEX-SHRINK

This property allows a flex-item to shrink if necessary, the default value set to flex-shrink is 1, so if we have a flex container with 5 items, and we set item4 to flex-shrink:2, flex item4 will shrink 2x (twice) as fast as the other items, if we set it to 3, it would shrink 3x as fast

FLEX-BASIS

The flex-basis property defines the initial size of an item inside of a container, BEFORE the extra space is distributed, this is used instead of the 'width' property, use flex-basis to set the initial width of the flex-item, flex-basis can use pixel values, percentages, or relative units, and it's default value is 'auto'

FLEX

This is a shorthand for all 3 (flex-grow, flex-shrink, and flex-basis). If we have: .item{ flex-grow: 2; flex-shrink: 5; flex-basis: 200px} we could just use flex and specify it like: .item{ flex: 2 5 200px}, and by default the flex is set to: .item{ flex: 0 1 auto;}

Align-self

This property is used to specify alignment of individual flex items
