So what is flexbox?

The CSS flexible box module aka FLEXBOX is a one-dimensional layout model that makes it easy to:

  • Design flexible and efficient layouts
  • Distribute space among items
  • Control their alignment in a given container

Before flexbox there were four layout modes:

  • BLOCK - For sections on a webpage
  • Inline - For text
  • Table - For two-dimensional table data
  • Positioned - For explicit position of an element

But these layouts did not provide enough flexibility, we used to have to use all sorts of 'hacks' to get the layout the way we wanted, working with FLOATS, trying to VERTICALLY ALIGN the content, creating EQUAL HEIGHT COLUMNS, kind've a nightmare

BOOTSTRAP is build on top of the flex layout, and FLEXBOX itself is crucial to know for front-end developers in order to style pages responsively

Why flexbox?

  • Lot's of flexibility
  • Arrange items more efficiently
  • Provide superior spacing
  • Help maintain proper alignment and order of items

TERMINOLOGY:

When we talk about flexbox, we mainly mean two entities:

  • FLEX-CONTAINER: Our main flex space
  • FLEX-ITEMS: The immediate children elements of the flex container

For ex: if we have 'div class='container' and inside we have < div > item 1 < / div > the 'item 1' is the flex-item, and the div class is our flex container

Now we also deal with FLEXBOX AXES

  • MAIN AXIS, or X AXIS WHICH RUNS LEFT TO RIGHT ON OUR SCREEN
  • CROSS AXIS, or Y AXIS WHICH RUNS TOP TO BOTTOM ON OUR SCREEN

The starting point of the main axis is titled: MAIN START

The end point of the main axis is titled: MAIN END

And the length from the main start to the main end is called: MAIN SIZE

So in general we say that flex items FLOW from main start to main end and take up the main size as the length

Similarly with the cross axis we have the top which is called: CROSS START

The bottom of the axis called: CROSS END

And the distance between them is known as the: CROSS SIZE

Understanding flexbox is getting familiar with the properties associated with the flex container and the flex items, it's like learning anything, learn what properties exist, and how they work / fit together

So let's now go over all the properties that can be applied to a flex container:

  • DISPLAY: The display property defines the flex container, and is mandatory to work with flexbox
  • FLEX-DIRECTION: This defines the direction in which the flex-items are placed within the container
  • FLEX-WRAP: This is used to control the wrapping of items in a container
  • FLEX-FLOW: This is used as a shortcut for combining FLEX-WRAP & FLEX-DIRECTION
  • JUSTIFY CONTENT: Defines the alignment of the items along the main (x) axis
  • ALIGN-ITEMS: Defines how items are laid out along the cross (y) axis
  • ALIGN-CONTENT: This is similar to justify-content, the difference being this will align along the cross axis, instead of the main axis, BUT ALIGN-CONTENT ONLY WORKS WHEN THERE ARE MULTIPLE ROWS OF FLEX-ITEMS IN THE CONTAINER
